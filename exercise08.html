jQuery - JavaScript library

JavaScript is the most widely-used language for adding dynamic behavior to web pages. The JavaScript community contributes to a collection of libraries that extend and ease its use. In this course, you will learn about jQuery, a JavaScript library that makes it easy to add dynamic behavior to HTML elements.

Example login functionality

const login = document.getElementById('login');
const loginMenu = document.getElementById('loginMenu');

login.addEventListener('click', () => {
  if(loginMenu.style.display === 'none'){
    loginMenu.style.display = 'inline';
  } else {
    loginMenu.style.display = 'none';
  }
});

jQuery makes your life simpler:

$('#login').click(() => {
  $('#loginMenu').toggle()
});

jQuery methods

The pre-made Lego structures are like jQuery methods. You can use these methods to add dynamic behavior, such as .hide(), .show(), .fadeIn(), .fadeOut() etc., to HTML elements.

It has become common practice to link the main JavaScript file at the bottom of the HTML document because a good deal of the content of the script will require that the dependencies, style sheets and elements exist before the browser can run the JavaScript that uses and references those things.

How to add jQuery

You can load jQuery using the respective CDN.

.ready()

a web page must be rendered in a user's browser before it's possible to have any dynamic behavior. To solve this problem, we will use our first jQuery method .ready().

When a web page is loaded, the browser creates a Document Object Model DOM of the page.

DOM

You should wrap all JavaScript behavior inside of the .ready() method. This will make sure the web page is rendered in the browser before any jQuery code executes in main.js. 

$(document).ready(() => {      
            
});

Document is the query object in this case. Typically, string, class, tag or ID is used. Here is a class example:

$('.someClass').handlerMethod();

Developers often save jQuery objects in variables, like so:

const $jQueryObject = $('.someClass');

It's best practice to start jQuery variables with an $-sign.

Event Handlers - .on()

The jQuery .on() method adds event handlers to jQuery objects. The method takes two parameters: a string declaring the event to listen for (the handler) and a callback function to fire when the event is detected.

$('#login').on('click', () => {
  $loginForm.show();
})

jQuery effects

Use the .toggle function to combine .show and .hide

  $('.toggle-button').on('click', () => {
    $('.first-image').toggle();
  });

With fadeIn() and fadeOut() you can hide and show with a time variable for effect.

 $('.fade-out-button').on('click', () => {
    $('.fade-image').fadeOut(500);
  });

Mouse events

event handler = event listener + callback function

$('.example-class').on('click', () => {
    // Execute code here when .example-class is clicked
});

The first argument is the event listener, the second (empty) one is the callback function.

The mouseenter event is triggered by entering a class with the cursor.

$('.menu-button').on('mouseenter', () => {
    	$('.nav-menu').show();
  });

Mousehide has the opposite effect.

$('.nav-menu').on('mouseleave', () => {
    $('.nav-menu').hide();
	});

It is also possible to chain multiple events.

$('.example-class').on('mouseenter', () => {
  $('.example-class-one').show();
}).on('mouseleave', () => {
  $('.example-class-one').hide();
});

But in order to address only the active element, use the current target selector:

  $('.product-photo').on('mouseenter', event => {
    $(event.currentTarget).addClass('photo-active')
  }).on('mouseleave', event => {
    $(event.currentTarget).removeClass('photo-active')
  })

For more information consult jqapi.

Style methods

To modify CSS properties of an element, jQuery provides a method called .css()

$('.example-class').css('color', '#FFFFFF');

You can also hand over value pairs.

{
  $('.nav-menu').on('mouseleave', () => {
    $('.nav-menu').hide();
    $('.menu-button').css({
      color: '#EFEFEF',
      backgroundColor: '#303030',
    });
  })
}

.Animate()

The first argument passed to .animate() is a JavaScript object of CSS property/value pairs that represent an element's end state. The second argument is the time to reach the end state.

$('.example-class').animate({
  height: '100px',
  width: '100px',
  borderWidth: '10px'
}, 500);

.addClass()

Avoid letting your code get to messy. Group the style code in the css file and add or remove the class from the element. This principle is called separation of concerns.

Separation of concerns is a design principle stating that code should be separated based on its purpose in a program.

$('.example-class').addClass('active');

In this case, the 'active' class (no period!) is added to the .example-class.

.removeClass()

You can also hide classes using the remove function.

$('.nav-menu').removeClass('nav-menu');

.toggleClass()

Use the toggle button to show and hide or add and remove.

$('.nav-menu').toggleClass('hide')

Document object model (DOM)

The DOM is a tree of objects based on the HTML document that is created by the browser when it loads a page. Every element in this page exists on some branch of the tree.

DOM - Document object model 
<div class='parent'>
  <div class='child' id='sibling1'>
    <p class='grandchild'></p>
  </div>
  <div class='child' id='sibling2'></div>
  <div class='child' id='sibling3'></div>
</div>

Understanding how elements relate to each other in the DOM makes it easy to efficiently select elements. We have covered several methods in this lesson including:

  • .children() to target an element's child elements.
  • .siblings() to target elements adjacent to an element.
  • .parent() to target an element's parent.
  • .closest() travels up the DOM tree from the current element to target the closest element with a given selector.
  • .next() to target the element immediately following the selected element.
  • .prev() to target the element that is immediately preceding the selected element.
  • .find() to target descendant elements by some selector, ie- class, id, tag etc.
