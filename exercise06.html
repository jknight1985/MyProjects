Command line

I followed the command line tutorial in codecademy, which I more and more find a useful resource for self-paced learning.

Navigating the file system

Additional input, like -a are called options. Theses options specify an order. Use cd to change the directory (one level deeper) and cd ..to go up.

  • ls -a lists all contents of a directory, including hidden files and directories
  • ls -l lists all contents in long format
  • ls -t orders files and directories by the time they were last modified
  • pwd - print working directory
  • Multiple options can be used together, like ls -alt

Manipulate files

  • From the command line, you can also copy, move, and remove files and directories:
  • cp copies files
  • mv moves and renames files
  • rm removes files
  • rm -r removes directories

Redirecting input and output

In this lesson, we'll focus on input and output (I/O) redirection. You can combine input and output from different programs.

$ echo "Hello" --> Hello

Standard input - stdin

Standard output - stdout

Standard error - stderr

The > redirectes a stout to a file, in this case hello.txt.

$ echo "Hello" > hello.txt

The cat command outputs the file. If you use > on0 a file, the file on the right hand side is overwritten.

>> add to the existing file content.

$ cat glaciers.txt >> rivers.txt

< inputs the content on the right and inputs it into the program on the left, e. g. cat.

| is the so-called pipe. Pipes standard output from left to the standard input on the right.

sort orders alphabetically.

uniq removes doubles.

grep stands for "global regular expression print". It searches for matching pattern, e. g. words within a file. grep -i makes it case insensitive.

sed stands for "stream editor". It is used to search and replace content.

Configuring the environment

Using nano as a command line editor. ~/.bash_profile is the name of file used to store environment settings. The ~ represents the user's home directory. The . indicates a hidden file.

source ~/.bash_profile

The source command activates the changes.

Aliases are shorcuts, e. g. alias pd="pwd". They need to be stored in the environmental variables.

alias ll="ls -la"
export USER="Jane Doe"
  • The line USER="Jane Doe" sets the environment variable USER to a name "Jane Doe". Usually the USER variable is set to the name of the computer's owner.
  • The line export makes the variable to be available to all child sessions initiated from the session you are in. This is a way to make the variable persist across programs.
  • At the command line, the command echo $USER returns the value of the variable. Note that $ is always used when returning a variable's value. Here, the command echo $USER returns the name set for the variable.

echo $HOME returns the HOME variable. It displays the path of the home directory.

$PATH called with echo lists a number of directories separated by a colon. For example, many commands we've learned are scripts stored in the /bin directory.

/bin/pwd
is the pwd script.

env is the environment command.
